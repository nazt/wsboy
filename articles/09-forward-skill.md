# /forward — ส่งต่อให้ตัวเองในอนาคต

## มันคืออะไร?

ก่อนปิด Claude ใช้ `/forward` เพื่อ**จดว่าต้องทำอะไรต่อ**

เหมือนเขียนโน้ตให้ตัวเองในอนาคต

---

## ใช้ยังไง?

```
/forward
```

Claude จะสร้างไฟล์ handoff ที่มี:
- สิ่งที่ทำไปแล้ว
- สิ่งที่ค้าง
- ขั้นตอนถัดไป

---

## ได้อะไรออกมา?

```markdown
# Handoff: แก้หน้า Portfolio

**วันที่**: 2026-01-20 18:30

## ทำไปแล้ว
- แก้ CSS ตรง header
- เพิ่มรูป project ใหม่

## ค้างอยู่
- [ ] ยังไม่ได้ optimize รูป
- [ ] link เสียอยู่ 2 ที่

## Session หน้า
- [ ] รัน image optimizer
- [ ] เช็ค link ทุกหน้า
```

---

## Flow การใช้งาน

```
ทำงานเสร็จ
    ↓
/rrr          ← สรุปงาน + บทเรียน
    ↓
/forward      ← จดว่าต้องทำอะไรต่อ
    ↓
ปิด Claude
    ↓
วันต่อไป...
    ↓
/recap        ← อ่านว่าค้างอะไร
```

---

## ต่างจาก /rrr ยังไง?

| /rrr | /forward |
|------|----------|
| **มองย้อนหลัง** | **มองไปข้างหน้า** |
| "ทำอะไรไปบ้าง" | "ต้องทำอะไรต่อ" |
| สรุป + บทเรียน | To-do list |
| เก็บไว้ใน memory | เก็บไว้ใน inbox |

---

## สำหรับ Artist

เหมือนตอนวาดรูปค้างไว้ แล้วเขียนโน้ตแปะบน canvas ว่า:

> "พรุ่งนี้: ลงสี background, แก้มือซ้าย, เช็ค proportion"

`/forward` ทำแบบเดียวกัน แต่ให้ Claude จดแทน

---

## เมื่อไหร่ควรใช้?

| สถานการณ์ | ใช้ /forward |
|----------|-------------|
| กำลังจะปิด Claude | ใช่ |
| ทำงานค้างไว้ | ใช่ |
| ต้องพักยาว | ใช่ |
| ทำเสร็จหมดแล้ว | ไม่จำเป็น |

---

## Tips

1. ใช้คู่กับ `/rrr` เสมอ
2. `/rrr` ก่อน → `/forward` ทีหลัง
3. session หน้าใช้ `/recap` เปิดอ่าน
